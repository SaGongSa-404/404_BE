- 선행 PR: 없음
- 후속 PR: 이 기준선 위에서 기능별 구현 PR 진행
- 참고: 현재 `develop`에는 Spring Boot 앱 기본 구조가 거의 없어 Gradle 설정, 애플리케이션 엔트리포인트, JPA/Flyway/PostgreSQL 설정은 함께 포함했습니다.

### 이 PR에서 변경한 것
- JPA, Flyway, PostgreSQL 기반의 도메인/스키마 베이스 설정을 추가했습니다.
- MVP 기준 핵심 도메인 엔티티와 enum을 `Auth`, `User`, `Budget`, `Item`, `Decision`, `Reflection`, `Notification`, `Mascot`, `Social`, `Terms/Consent` 영역으로 정리했습니다.
- 기획팀 Q&A 반영 문서 기준으로 초기 스키마 마이그레이션을 수정했습니다.
- PostgreSQL 기준으로 `jsonb` 컬럼과 soft delete/cancel 대상 partial index를 반영했습니다.
- `YYYY-MM` 예산 사이클, 셀프체크 Yes 개수/판별 결과, 문항별 단일 답변 같은 도메인 불변조건을 DB 제약으로 반영했습니다.
- 금액/횟수/투표 수의 음수 방지, 비율/신뢰도 범위, 7일 후 후회 알림의 구매 결정 연결 같은 DB 제약을 보강했습니다.
- 탈퇴 시각, 알림 읽음 시각, 리마인더 발송/취소 시각이 상태값과 어긋나지 않도록 DB 제약을 추가했습니다.
- Testcontainers PostgreSQL 기반 스모크 테스트로 실제 마이그레이션과 JPA 검증을 수행합니다.
- 재결정 이력, 합리성 판별 결과, 푸시 알림 설정/기기 토큰, 투표/댓글, 약관 버전/동의 이력을 스키마에 반영했습니다.
- 애플리케이션과 테스트 환경에서 스키마 검증이 자동으로 일어나도록 설정했습니다.
- 스키마 스모크 테스트를 추가해 주요 테이블과 기획 반영 핵심 컬럼이 실제로 생성되는지 확인했습니다.
- 패키지명과 프로젝트명을 위굴 백엔드 기준으로 정리했습니다.
