# 에이전트 세팅 원칙 검토 메모

작성일: 2026-03-19
목적: 이후 워크플로 대화에서 사용할 기준선을 명확히 한다.

## 1. 하나의 에이전트는 하나의 작업을 해야 한다

판정: `조건부 동의`

왜 그렇게 보나:
- 주 책임이 하나일수록 품질과 책임소재가 명확해진다.
- 에이전트가 여러 목적을 동시에 가지면 산출물이 흐려지고, handoff가 약해지기 쉽다.
- 특히 `아이디어 발굴`, `시장 검증`, `전략 보강`처럼 성격이 다른 일은 분리할수록 좋다.

예외:
- 작업이 아주 작고 문맥 전환 비용이 더 큰 경우
- 분리해도 얻는 이득보다 coordination cost가 큰 경우
- 같은 타입의 일 여러 개를 한 에이전트가 묶어서 처리하는 편이 더 자연스러운 경우

권장 해석:
- “하나의 에이전트 = 하나의 주 책임”이 맞다.
- 다만 주 책임 안에서 작은 하위 행동까지 모두 따로 떼라는 뜻은 아니다.

## 2. 자신의 작업 범위를 넘어서지만 하나의 작업이라고 할 만큼은 아닌 것은 서브에이전트로 구동한다

판정: `동의`

왜 그렇게 보나:
- 주 책임은 유지하되, 보조성 조사/정리/비교 작업을 서브에이전트로 떼면 속도가 빨라진다.
- 예를 들어 2번 에이전트가 시장 검증을 하면서 특정 vertical의 경쟁사 리서치만 따로 떼는 건 좋은 패턴이다.
- 이런 구조는 main agent의 판단 품질을 유지하면서 병렬성을 얻는다.

예외:
- 서브에이전트를 붙이는 순간 오히려 handoff 문서가 더 길어지는 경우
- 하위 작업이 너무 작아서 직접 처리하는 편이 빠른 경우
- 결과가 강하게 상호의존해 병렬 분리가 잘 안 되는 경우

권장 해석:
- “주 책임 유지 + 보조 작업 분리”가 가장 좋은 패턴이다.
- 서브에이전트는 main agent의 판단을 대체하는 용도가 아니라 보강하는 용도로 써야 한다.

## 3. 에이전트의 작업과 관련된 모든 기록은 md 파일 산출물로 만드는 게 유리하다

판정: `조건부 동의`

왜 그렇게 보나:
- md 파일은 회차 추적, 비교, 공유, 재사용에 유리하다.
- 특히 멀티에이전트 구조에서는 나중에 “왜 이 판단을 했는지”를 다시 보기 쉽다.
- 최종 보고서, 회차 요약, 평가 문서는 md로 남기는 것이 확실히 좋다.

하지만 주의할 점:
- “모든 기록”을 다 md로 남기면 문서 폭증이 생긴다.
- 중간 생각, 임시 메모, 중복 조사까지 다 파일화하면 최신본 찾기가 더 어려워진다.
- 중요한 건 `모든 로그`가 아니라 `의사결정 로그`를 남기는 것이다.

예외:
- 아주 짧은 일회성 확인
- 나중에 다시 볼 가치가 거의 없는 임시 상태
- 이미 상위 종합 문서에 흡수된 중복 기록

권장 해석:
- 모든 작업 기록을 남기기보다, `중요한 판단/회차 결과/최종 산출물`만 md로 남기는 것이 좋다.

## 4. 최종 권장 운영 원칙

1. 에이전트 하나당 주 책임 하나
2. 작은 보조 작업은 서브에이전트로 분리
3. 모든 로그가 아니라 중요한 의사결정 로그만 md로 남기기
4. 파일은 초안/검토본/최종본을 구분하기
5. 취합 문서는 항상 읽는 순서와 최신본을 먼저 안내하기

## 5. 짧은 결론

- 첫 번째 주장: `조건부 동의`
- 두 번째 주장: `동의`
- 세 번째 주장: `조건부 동의`

가장 실무적인 형태로 바꾸면 이렇게 정리할 수 있다.

> 하나의 에이전트는 하나의 주 책임을 맡고, 그 책임을 돕는 작은 하위 작업만 서브에이전트로 분리한다.  
> 기록은 전부 남기기보다 중요한 판단과 결과만 md 산출물로 남긴다.
