# Accuracy Remediation Plan

작성일: 2026-03-22

## 목표

- 현재 시스템의 분류/판정 정확도를 `300개 샘플 기준 90%`까지 끌어올린다.
- 링크 수집은 하지 않는다.
- 개선은 `입력 필터 -> 추출 -> 규칙 -> AI 게이트` 순서로 쌓는다.
- 먼저 빈도가 높고 오분류 비용이 큰 유형부터 고친다.

## 우선순위 원칙

1. `입력 필터`가 흔들리면 이후 단계가 모두 흔들린다.
2. `추출`이 부정확하면 규칙과 AI 게이트가 과적합된다.
3. `규칙`은 빠르고 안정적인 1차 방어선이어야 한다.
4. `AI 게이트`는 마지막 보정층으로만 써야 한다.
5. 유형별로 하나의 공통 파이프라인을 강요하지 말고, 판정 신호의 가중치를 다르게 둔다.

## 문제 유형 우선순위

### P0. 일반 유튜브 정보형

- 특징:
  - 가장 범용적이고 샘플 수가 많다.
  - 제목이 넓고 설명이 빈약해 오분류가 쉽게 난다.
- 주된 실패:
  - 정보형인데 리뷰/가이드/브랜딩 콘텐츠로 섞임
  - 영상 제목의 키워드만 보고 과대판정함
- 개선 포인트:
  - 입력 필터:
    - `how to`, `what is`, `explained`, `tips`, `guide` 같은 정보 의도를 우선 감지
    - 채널 도메인보다 영상 자체의 질문형 패턴을 먼저 본다
  - 추출:
    - 제목, 설명 첫 줄, 자막 초반 30초, 썸네일 텍스트를 동시에 본다
    - 질문-답변 구조가 있으면 정보형 가중치를 올린다
  - 규칙:
    - 정보 의도가 명시적이면 다른 유형보다 먼저 통과시킨다
    - 제품명, 장소명, 캠페인명이 있어도 정보 의도가 강하면 정보형 유지
  - AI 게이트:
    - 짧은 설명만으로 확신이 낮을 때만 호출한다
    - 출력은 `정보형 / 비정보형` 2분류 보정에 집중한다

### P1. Explainer

- 특징:
  - 개념 설명, 원리 설명, 비교 설명이 섞여 있다.
  - 일반 정보형과 가장 많이 충돌한다.
- 주된 실패:
  - 짧은 정보형과 구분이 안 됨
  - 단순 소개 영상인데 explainer로 오판
- 개선 포인트:
  - 입력 필터:
    - `explained`, `explainer`, `deep dive`, `vs`, `comparison`, `why it matters` 신호를 분리
    - 개념어 밀도가 높을수록 explainer 후보로 올린다
  - 추출:
    - 정의문, 원인-결과문, 비교문, 도식형 표현을 추출
    - 자막 초반에 개념 정의가 나오는지 본다
  - 규칙:
    - `개념 설명 + 구조화된 전개 + 요약`이 모두 있으면 explainer 우선
    - 단일 팁 나열형이면 일반 정보형으로 되돌린다
  - AI 게이트:
    - 일반 정보형과 경계가 애매할 때만 `explainer 여부`를 판정하게 한다
    - 판정 이유에 `정의/비교/원리` 중 무엇이 있었는지 반드시 남긴다

### P1. Travel Guide

- 특징:
  - 지역성, 계절성, 일정성, 실사용 정보가 중요하다.
  - 정보형과 형태가 비슷하지만 맥락 신호가 다르다.
- 주된 실패:
  - 도시명만 보고 여행 가이드를 놓침
  - 관광/맛집/숙소/교통 정보가 섞여 있어 분류가 흔들림
- 개선 포인트:
  - 입력 필터:
    - `travel`, `trip`, `itinerary`, `things to do`, `best place`, `must visit` 같은 여행 의도를 우선 감지
    - 지역명, 시즌명, 공항/역/숙소/교통 용어를 강한 보조 신호로 본다
  - 추출:
    - 장소명, 동선, 일정 단위, 비용 단위, 이동 수단을 추출
    - 지도/루트/일정형 표현을 별도 신호로 분리
  - 규칙:
    - 지역명만 있는 콘텐츠는 travel guide로 자동 승격하지 않는다
    - 최소 2개 이상의 여행 실행 신호가 있어야 한다
  - AI 게이트:
    - 지역 정보가 있으나 콘텐츠 목적이 불명확할 때만 호출
    - 출력에서 `실용 여행 정보`와 `일반 지역 소개`를 구분하게 한다

### P2. Product Review

- 특징:
  - 구매 의도와 비교 신호가 강하다.
  - 일반 정보형, explainer와 제목 패턴이 많이 겹친다.
- 주된 실패:
  - 사용기와 단순 언박싱을 섞음
  - 광고성 소개를 실제 리뷰로 오판
- 개선 포인트:
  - 입력 필터:
    - `review`, `best`, `vs`, `worth it`, `unboxing`, `honest opinion` 신호를 우선 감지
    - 가격, 모델명, 스펙, 대안 비교가 있으면 리뷰 후보로 올린다
  - 추출:
    - 장단점 언급, 비교 대상, 실사용 기간, 결론 문장을 추출
    - 광고 문구와 실제 평가 문장을 분리한다
  - 규칙:
    - 단순 소개/협찬 고지만 있으면 review로 확정하지 않는다
    - 비교 또는 평가 결론이 있어야 product review로 인정한다
  - AI 게이트:
    - 협찬성 콘텐츠와 실제 리뷰가 혼재할 때만 호출
    - `평가`, `비교`, `추천 여부`가 명시되었는지 확인시킨다

### P2. Marketing Guide

- 특징:
  - 실행 지침, 프레임워크, 성장 팁이 중심이다.
  - 정보형과 겹치지만 행동 유도 신호가 더 강하다.
- 주된 실패:
  - 일반 비즈니스 정보나 동기부여 영상으로 흘러감
  - 실전 가이드와 개념 설명을 구분하지 못함
- 개선 포인트:
  - 입력 필터:
    - `marketing`, `growth`, `strategy`, `campaign`, `funnel`, `lead`, `conversion` 신호를 우선 감지
    - 타깃 고객, 채널, 예산, KPI가 보이면 우선순위를 높인다
  - 추출:
    - 실행 단계, 체크리스트, KPI, 채널명, 예산 관련 표현을 추출
    - 사례보다 실행 구조가 있는지를 본다
  - 규칙:
    - 단순 개념 소개는 marketing guide로 올리지 않는다
    - 실행 단계가 2개 이상이면 guide 성격을 강화한다
  - AI 게이트:
    - 전략론과 실행 가이드가 섞였을 때만 사용
    - `실행 가능성`과 `추상성`을 분리해 판단하게 한다

### P3. Social Design

- 특징:
  - 썸네일, 배너, 포스터, 카드, SNS 이미지 등 시각 산출물이 중심이다.
  - 텍스트보다 이미지/디자인 맥락이 중요하다.
- 주된 실패:
  - 일반 콘텐츠 설명과 디자인 의도를 혼동
  - 소셜용 산출물인데 정보형 텍스트만 보고 놓침
- 개선 포인트:
  - 입력 필터:
    - `instagram`, `youtube thumbnail`, `banner`, `post`, `flyer`, `design`, `template` 같은 디자인 의도 신호를 감지
    - 플랫폼명과 시각 산출물 용어가 함께 있으면 우선 후보로 올린다
  - 추출:
    - 이미지 내 텍스트, 레이아웃 키워드, 비율, 포맷, CTA 문구를 추출
    - 본문 설명보다 썸네일/이미지 속 문구를 더 강하게 본다
  - 규칙:
    - 단순 홍보 문구만 있고 실제 디자인 산출물 신호가 없으면 보수적으로 판정
    - 포맷 신호가 있으면 일반 마케팅 정보보다 social design을 우선 검토
  - AI 게이트:
    - 이미지 중심 콘텐츠에서만 제한적으로 사용
    - 텍스트만으로 판단이 어려울 때 시각 포맷 여부를 확인하게 한다

## 레이어별 개선 과제

### 1) 입력 필터

- 목적:
  - 애초에 잘못된 유형 후보를 줄인다.
- 해야 할 일:
  - 정보형, explainer, travel, review, marketing, design 의도 키워드 사전을 분리한다
  - 플랫폼/지역/상품/시각 포맷 신호를 함께 본다
  - 한 문장에 여러 의도가 있을 때 주 의도를 먼저 잡는다
- 성공 기준:
  - 명백한 오분류 후보가 추출 단계로 내려오는 비율을 줄인다

### 2) 추출

- 목적:
  - 규칙과 AI가 볼 수 있는 신호를 표준화한다.
- 해야 할 일:
  - 제목, 설명, 자막 초반, 썸네일 텍스트, 엔티티를 공통 스키마로 추출한다
  - 의도 신호, 실행 신호, 비교 신호, 시각 포맷 신호를 분리 저장한다
  - 추출 실패 시 빈 값이 아니라 `unknown`과 `missing reason`을 남긴다
- 성공 기준:
  - 동일한 콘텐츠에 대해 반복 실행했을 때 핵심 신호가 흔들리지 않는다

### 3) 규칙

- 목적:
  - 가장 흔한 패턴은 규칙으로 빨리 고정한다.
- 해야 할 일:
  - 유형별 최소 통과 조건을 둔다
  - 충돌 규칙을 둔다:
    - 정보형 vs explainer
    - travel guide vs 일반 지역 소개
    - review vs 단순 언박싱
    - marketing guide vs 일반 정보
    - social design vs 일반 홍보 이미지
  - 점수 기반으로 애매한 케이스만 AI로 넘긴다
- 성공 기준:
  - 쉬운 샘플이 AI로 과도하게 넘어가지 않는다

### 4) AI 게이트

- 목적:
  - 애매한 샘플만 보정하고, 최종 정확도를 끌어올린다.
- 해야 할 일:
  - AI는 `최종 판정`보다 `경계 사례 판정`에 집중시킨다
  - 출력 형식을 고정한다:
    - 예측 유형
    - 근거 신호
    - 애매한 이유
    - 다음에 필요한 추가 신호
  - confidence가 낮으면 보수적으로 상위 공통 유형으로 되돌린다
- 성공 기준:
  - AI 호출 수는 줄고, AI가 개입한 샘플의 정답률은 오른다

## 실행 순서

1. P0 일반 유튜브 정보형부터 규칙을 고정한다.
2. P1 explainer와 travel guide의 경계 규칙을 분리한다.
3. P2 product review와 marketing guide의 구매/실행 신호를 정리한다.
4. P3 social design은 시각 포맷 신호가 안정화된 뒤 보강한다.
5. 매 단계마다 300개 샘플로 재측정하고, AI 게이트 비중이 늘지 않는지 확인한다.

## 최종 판정 기준

- 전체 정확도 90% 이상
- P0 유형에서 오분류가 가장 크게 줄어들 것
- 애매한 샘플만 AI 게이트로 넘어갈 것
- 규칙만으로 판정 가능한 샘플 비율이 계속 증가할 것
