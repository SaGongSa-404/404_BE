# A 평가 기반 개선된 워크플로 버전

작성일: 2026-03-19  
작성자 역할: B 에이전트  
기준 문서: `workflow_review_for_agentB_2026-03-19.md`

## 1. 개선 방향 요약

A 에이전트 평가를 바탕으로 현재 워크플로는 유지하되, 아래 4가지를 강화한다.
- 회차 종료 조건
- 폐기 기준
- 인계 포맷
- 산출물 구조

## 2. 개선된 운영 규칙

### 규칙 1. 하나의 에이전트는 하나의 1차 책임만 가진다
- 1번: 아이디어 발굴/수정
- 2번: 시장 검증/판정
- 3번: 전략/GTM 보강
- 취합: 최종 정리

작은 보조 작업은 허용되지만, 다른 에이전트의 핵심 책임까지 흡수하면 안 된다.

### 규칙 2. 회차 종료 조건을 명시한다
- 아래 3개 중 2개 이상 충족 시, 다음 루프 대신 PRD/MVP 단계로 승격
  - 한 문장 pain이 명확함
  - 첫 고객 채널이 구체적임
  - 6주 실험 KPI가 수치로 정의됨

### 규칙 3. 폐기 기준을 명시한다
- 아래 중 하나라도 강하게 해당되면 `폐기` 또는 `보류`
  - 반복 사용성이 약함
  - 기존 강자와 정면충돌하며 wedge가 없음
  - 6주 MVP로 가치 검증이 불가능함

### 규칙 4. 인계 포맷을 강제한다
- 1번 -> 2번
  - 한 문장 pain
  - 구매자
  - 실사용자
  - 반복 시점
  - 돈 낼 이유
  - 버린 범위
  - 6주 MVP
- 2번 -> 3번
  - 유사 서비스/대체재
  - 수요 강도
  - 돈 안 낼 이유
  - 치명 리스크 3개
  - 생존 판정
  - 살리는 조건
- 3번 -> 1번
  - 유지 강점
  - 버릴 요소
  - 첫 고객 채널
  - 6주 KPI
  - 수익화 방향
  - 수정 질문 2개 이상

## 3. 개선된 회차 운영 표준

### 1회차
- 목적: 존재 가능성 확인
- 결과: `살림 후보`와 `초기 폐기 후보` 분리

### 2회차
- 목적: 세그먼트와 wedge 고정
- 결과: 상위 절반만 남김

### 3회차
- 목적: 세일즈 메시지와 MVP 경계 확정
- 결과: PRD/MVP로 넘어갈 top3 결정

### 4회차 이상
- 목적: 상위권 아이디어만 정교화
- 규칙: 하위권 아이디어는 더 돌리지 않음

## 4. 개선된 산출물 구조

### 기본 구조
- `reports/idea_agent_report_3cycles_YYYY-MM-DD.md`
- `reports/loop10/aggregate_10cycles_report.md`
- `reports/recalibrated_scorecard_YYYY-MM-DD.md`

### 권장 회차 구조
- `reports/round4/agent1.md`
- `reports/round4/agent2.md`
- `reports/round4/agent3.md`
- `reports/round4/aggregate.md`

### 권장 원칙
- 종합본은 반드시 하나
- 회차별 로그는 반드시 agent1/2/3 분리
- 파일명에는 날짜 또는 회차가 반드시 들어갈 것

## 5. 개선된 최종 SOP

1. 1번 에이전트가 원본 아이디어 5~10개 발굴
2. 2번 에이전트가 전수 검증 후 `살림 / 조건부 살림 / 보류 / 폐기` 판정
3. 3번 에이전트가 조건부 살림 이상만 보강
4. 1번 에이전트가 상위 절반 수정안 작성
5. 2회차에서 세그먼트와 wedge 고정
6. 3회차에서 top3만 남기고 PRD/MVP 단계 승격 여부 판단
7. 취합 에이전트가 종합본과 점수표 작성

## 6. 개선 버전의 기대 효과

- 루프가 길어져도 관리 비용이 덜 커진다.
- 폐기 기준이 강해져 우선순위가 선명해진다.
- 에이전트 간 인계가 빨라진다.
- 나중에 문서를 다시 볼 때 구조가 덜 흐트러진다.

## 7. B 에이전트 결론

현재 워크플로는 기본 구조가 이미 강하다. 문제는 방향이 아니라 운영 마찰이다. 따라서 가장 큰 개선은 `더 복잡한 프레임워크 추가`가 아니라 `종료 조건, 폐기 기준, 인계 포맷, 파일 구조`를 엄격히 만드는 것이다.
